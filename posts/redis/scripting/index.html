redis scripting

  • 官方地址：https://redis.io/docs/latest/commands/?group=scripting
  • 分片集群中，要注意什么：
    • 跨分片操作：如果Lua脚本中的命令尝试访问不同分片上的键，Redis Cluster会返回一个错误，指出不能执行跨分片脚本。这意味着，Lua脚本在Redis Cluster中不支持跨分片的操作。
    • 错误处理：与事务类似，如果在Lua脚本执行过程中遇到转向错误（例如，因为键不在当前分片上），整个脚本会停止执行，并且已经执行的部分不会被回滚。

EVAL#

  1. 从 Redis 2.6.0 版本开始，通过内置的 Lua 解释器，可以使用 EVAL 命令对 Lua 脚本进行求值。
  2. script 参数是一段 Lua 5.1 脚本程序，它会被运行在 Redis 服务器上下文中，这段脚本不必(也不应该)定义为一个 Lua 函数。
  3. numkeys 参数用于指定键名参数的个数。
  4. 语法：
EVAL script numkeys [key [key ...]] [arg [arg ...]]
  1. 示例：
$ EVAL "return ARGV[1]" 0 hello
"hello"

$ eval "return {KEYS[1],KEYS[2],ARGV[1],ARGV[2]}" 2 key1 key2 first second
1) "key1"
2) "key2"
3) "first"
4) "second"

EVAL_RO#

  1. 这是 EVAL 命令的只读变体，无法执行修改数据的命令。
  2. 语法：
EVAL_RO script numkeys [key [key ...]] [arg [arg ...]]
  1. 使用示例：
$ SET mykey "Hello"
OK

$ EVAL_RO "return redis.call('GET', KEYS[1])" 1 mykey
"Hello"

$ EVAL_RO "return redis.call('DEL', KEYS[1])" 1 mykey
(error) ERR Error running script (call to b0d697da25b13e49157b2c214a4033546aba2104): @user_script:1: @user_script: 1: Write commands are not allowed from read-only scripts.

EVALSHA#

  1. 服务器使用 SCRIPT LOAD 命令缓存脚本。该命令在其他方面与 EVAL 相同。
  2. 语法：
EVALSHA sha1 numkeys [key [key ...]] [arg [arg ...]]
  1. 使用示例：
$ SCRIPT LOAD "return 'hello moto'"
"232fd51614574cf0867b83d384a5e898cfd24e5a"

$ EVALSHA "232fd51614574cf0867b83d384a5e898cfd24e5a" 0
"hello moto"

EVALSHA_RO#

  1. 这是 EVALSHA 命令的只读变体，无法执行修改数据的命令。
  2. 语法：
EVALSHA_RO sha1 numkeys [key [key ...]] [arg [arg ...]]

SCRIPT LOAD#

  1. 将脚本 script 添加到脚本缓存中，但并不立即执行这个脚本。
  2. EVAL 命令也会将脚本添加到脚本缓存中，但是它会立即对输入的脚本进行求值。
  3. 如果给定的脚本已经在缓存里面了，那么不做动作。
  4. 在脚本被加入到缓存之后，通过 EVALSHA 命令，可以使用脚本的 SHA1 校验和来调用这个脚本。
  5. 脚本可以在缓存中保留无限长的时间，直到执行 SCRIPT FLUSH 为止。
  6. 语法：
SCRIPT LOAD script
  1. 使用示例：
$ SCRIPT LOAD "return 'hello moto'"
"232fd51614574cf0867b83d384a5e898cfd24e5a"

$ EVALSHA "232fd51614574cf0867b83d384a5e898cfd24e5a" 0
"hello moto"

SCRIPT EXISTS#

  1. 给定一个或多个脚本的 SHA1 校验和，返回一个包含 0 和 1 的列表，表示校验和所指定的脚本是否已经被保存在缓存当中。
  2. 返回值：一个列表，包含 0 和 1 ，前者表示脚本不存在于缓存，后者表示脚本已经在缓存里面了。
  3. 语法：
SCRIPT EXISTS sha1 [sha1 ...]
  1. 使用示例：
$ SCRIPT LOAD "return 'hello moto'"    # 载入一个脚本
"232fd51614574cf0867b83d384a5e898cfd24e5a"

$ SCRIPT EXISTS 232fd51614574cf0867b83d384a5e898cfd24e5a
1) (integer) 1

$ SCRIPT FLUSH     # 清空缓存
OK

$ SCRIPT EXISTS 232fd51614574cf0867b83d384a5e898cfd24e5a
1) (integer) 0

SCRIPT FLUSH#

  1. 清除所有 Lua 脚本缓存。
  2. 语法：
    • ASYNC：异步刷新缓存
    • SYNC：同步刷新缓存
SCRIPT FLUSH [ASYNC | SYNC]
  1. 使用示例：
$ SCRIPT FLUSH
OK

SCRIPT KILL#

  1. 杀死当前正在运行的 Lua 脚本，当且仅当这个脚本没有执行过任何写操作时，这个命令才生效。
  2. 这个命令主要用于终止运行时间过长的脚本，比如一个因为 BUG 而发生无限 loop 的脚本，诸如此类。
  3. SCRIPT KILL 执行之后，当前正在运行的脚本会被杀死，执行这个脚本的客户端会从 EVAL 命令的阻塞当中退出，并收到一个错误作为返回值。
  4. 语法：
SCRIPT KILL
  1. 使用示例：
# 没有脚本在执行时
$ SCRIPT KILL
(error) ERR No scripts in execution right now.

# 成功杀死脚本时
$ SCRIPT KILL
OK
(1.30s)

# 尝试杀死一个已经执行过写操作的脚本，失败
$ SCRIPT KILL
(error) ERR Sorry the script already executed write commands against the dataset. You can either wait the script termination or kill the server in an hard way using the SHUTDOWN NOSAVE command.
(1.69s)
